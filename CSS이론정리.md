색상의 표현
(1) 영문색상명 표기 ( 예:red, blue)
(2) 색상코드 사용
	: 6개의 숫자(16진수)로 되어 있음.

컴퓨터는 RGB 색상모드(8bit)를 사용
Red, Green, Blue의 3원색으로 기반으로 한 색광혼합방식

한가지색상이 256개의 색상값을 가짐.
0 ~ 255(16진수변환: 00 ~ FF)

#FF0055
-> FF : Red값
-> 00 : Green값
-> 55 : Blue값

CSS문서에서는 #FF0055 -> #F05로 줄여서 사용가능하다.


■ CSS 사용하는 이유
	(1) 표현할 수 있는 스타일이 다양하고, 세분화 되어있다.
		- 동적인 표현과 반응형 표현까지 가능해
	(2) 구조와 스타일을 분리하여, 작성할 수 있다.

■ CSS의 다양한 정의방법 = 어디에다 작성하는가? (p96~99)
	(1) CSS파일 : 연결하는 모든 웹문서가 스타일을 공유
		① html안에서는 <link>태그를 사용하여 연결
			- href속성 : 연결하는 css문서를 지정한다.
			- rel속성 : 연결하는 대상의 타입을 지정한다.
		② 스타일시트안에서는 @import url("주소"); 를 사용하여 연결
	(2) <style>태그 : 해당 문서안에서 여러요소가 스타일을 공유
	(3) style속성(인라인스타일) : 해당 요소에만 스타일을 적용함.

■ 셀렉터(=선택자)의 기본유형 (p100-p103)
	:셀렉터란, CSS에서 대상을 구성하는 요소
	(1) 태그셀렉터 : 태그를 대상으로 스타일을 부여
	(2) 아이디셀렉터(#아이디명) : 아이디를 대상으로 스타일을 부여
		- id속성은 중복불가이다. 문서안에서 동일한 아이디값을 가지는 요소가
		있으면 안된다.
	(3) 클래스셀렉터(.클래스명) : 클래스를 대상으로 스타일을 부여
		- class속성은 id속성과 다르게, 중복부여가 가능하고,
		여러 클래스값을 가지는 다중부여도 가능하다.
	※ 아이디명과 클래스명 부여시, 주의사항
		① 숫자로 시작하면 안되고, 숫자로만 되어 있어도 안된다.
		② 띄어쓰면 안된다.(클래스의 경우, 다중부여로 처리된다.)
		③ 대소문자를 구분하여 셀렉터에 적용된다.

■ CSS 우선순위 : 스타일의 적용순서(p119)
	- 우선순위가 높다. = 최종적으로 적용되는 스타일을 뜻함.
	(1) 작성방식 상관없이, 작성순서가 나중에 작성된 스타일이 우선순위가 높다.
	(2) 1번보다, 대상을 구성하는 셀렉터의 수가 더 많은 쪽이 우선순위가 높다.
	(3) 태그셀렉터보다 클래스셀렉터가, 클래스셀렉터보다 아이디셀렉터가
		우선순위가 더 높다.
	(4) 인라인스타일이 우선순위가 가장 높다.
	※[!important] 속성값 : 속성에 해당 속성값을 추가하면,
		우선순위를 무시하고, 해당 속성의 스타일을 강제할 수 있다.
		- [!important] 특성이 중복인 경우, 우선순위를 따른다.




■ 태그의 구조
(1) contents영역 : 태그안
	- 컨텐츠가 표시되는 영역, 텍스트나 또 다른 태그가 표현되는 영역
	- width와 height 속성에 의해 크기조절됨.
(2) padding영역 : 내부여백 or 영역의 확장
	- 컨텐츠영역과 테두리 사이 간격
	- background만 표시됨.
(3) border영역 : 테두리

(4) margin영역 : 외부여백
	- 해당 요소(태그)와 외부 요소들과의 간격(테두리 기준)
	- 다른요소와의 margin영역은 서로 겹친다.
	- 마이너스값사용 : 상대적위치라고 해석할 수 있다.

■ 태그의 유형(★★★★★)
1. block 태그 : 사이트의 구조를 만들때 사용됨, 영역/컨텐츠영역
	- div, h1~h6,  p, ul, ol, li, dl, dt, dd, hr, address,
		header, main, footer, nav, aside, section, article
	(1) 수직정렬  (2)크기조절이 가능함.
	(3) 기본크기 : 너비는 부모와 같고, 높이는 내용만큼 설정된다.

2. inline 태그 : 텍스트나 내용을 감싸는 태그
	- a, span, strong, em, (i, b, small, del, ins, sup, sub, mark,)
	(1) 수평정렬  (2)크기조절이 불가능함.
	(3) 기본크기 : 너비는 내용만큼, 높이는 글자크기(font-size)만큼 설정된다.
	(4) 공백(띄어쓰기)를 인식함.
	(5) 블럭요소를 감쌀 수 없다.(블럭요소의 부모(조상)요소가 될 수 없다.)
	(6)

3. inline-block 태그 : 컨텐츠에 해당되는 태그(텍스트와 같다)
	- img, iframe, video, audio
	(1)수평정렬  (2)크기조절이 가능함.
	(3)기본크기 : 너비와 높이 둘다 내용만큼, 텍스트를 감싸는 경우, line-height만큼
							설정된다.
	(4) 공백(띄어쓰기)을 인식함.  (5)블럭요소를 감쌀 수 있다.

■ 블록요소를 수평하게 나열하는 방법
	1. display:inline; (inline-block;)
		[유의사항과 특징]
		(1) 크기조절이 안됨.(inline만 해당)
		(2) 공백(띄어쓰기) 인식함.
		(3) 가운데정렬가능(text-align:center;)

	2. float:left; (right;) 
		[유의사항과 특징]
		(1) 가운데정렬이 안됨.
		(2) 줄바꿈 : clear 속성을 사용하여, 줄바꿈
		(3) block요소가 float 적용된 요소들을 인식하지 못함. - ★★★
			[float요소를 인식하는 방법]
			(case1) float요소 다음에 block요소가 오는 경우
				: clear속성을 사용하여, 선행되는 float효과를 지운다.
			(case2) float요소를 감싸는 부모요소의 경우
				① 부모요소에게도 float를 설정한다.(추천하지 않음.)
				② 필요한 만큼의 높이값을 강제적으로 부여한다.(유동적이지 않음.)
				
				
				
				
				③ overflow:hidden;을 설정한다. -  높이를 고정하면 안됨.
				
				
				
				
				
				④ float를 인식하는 마지막 자식요소(block)를 추가한다.
				.clearfix::after{content:""; display:block; clear:both;}

		※ clear:both; (left; right;)
			: 선행되는 float효과를 지운다.
			(이전 요소와 같이 수평정렬이 되지않음. - 줄바꿈)



■ 가상클래스(유사클래스) -" : "
	: 특정한 행위나 상태에 따라 스타일의 대상으로 지정함.
	
	□ 자식요소에 따라 가상클래스 지정
	:first-child { 첫번째 자식요소가 스타일의 대상이 됨. }
	:last-child { 마지막 자식요소가 스타일의 대상이 됨. }
	:nth-child(♣) { ♣번째 자식요소가 스타일의 대상이 됨. }
	:nth-child(♣n) { ♣의 배수 번째 자식요소가 ..... }
		(예) li:nth-child(3n) { 3,6,9,12,15,18, ...  번째 자식요소 모두  }
	:nth-child(♣n+♥) { ♥번째 요소부터 ♣의 배수로 증가하는 모든 자식요소..}
		(예) li:nth-child(4n+3) { 3,7,11,15,19, .... 번째 자식요소 모두}	
	※ ♣,♥ 는 숫자(양의 정수)만 입력한다.
	※ "n"은 0부터 시작하는 자연수가 대입된다.
	:nth-child(odd) {홀수번째 자식요소에게 스타일을 부여함.}
	:nth-child(even) {짝수번째 자식요소에게 스타일을 부여함.}
	
	□ 링크와 마우스 동작에 의한 가상클래스
	a:link { 링크가 설정된 경우, 스타일을 부여 }
	a:visited { 방문한 링크인 경우, 스타일을 부여 }
	a:focus{ 탭에 의해 버튼이 선택된 경우, 스타일을 부여 }
	a:hover { 마우스를 올렸을 때, 스타일을 부여 } - ★
	a:active { 클릭하는 순간, 스타일을 부여 }
	※ [:link]와  [:visited]는 링크가 설정된 a태그에서만 적용됨
	※ [:focus]는 링크가 설정된 a태그와 폼요소
	※ [:hover]와 [:active]는 다른 태그에서도 적용가능함.- ★
	
	□ 체크박스와 라디오버튼이 선택(체크) 되었을 경우의 가상클래스
	:checked { 선택된 경우 . }

■ 포지션 속성 학습하기
position : 위치를 지정하는 방식(기준)을 설정된다.
	:static(기본값) - block은 수직하게, inline은 수평하게 나열되는 방식
	:absolute - 상위포지션을 기준으로 하여 위치가 지정되고,
		상위포지션이 없는 경우, 화면을 기준으로 하여 위치가 지정된다.
	:relative - 현 위치(자기자신)를 기준으로 하여 위치가 지정된다.
		주로, 상위포지션의 용도로 많이 사용된다. 
	:fixed - 화면을 기준으로 하여 위치가 지정되고, 화면에 고정이 된다.
	:sticky -  상위포지션기준 위치가 지정되고, 임계값이 넘어가면,
		화면에 고정된다.(되도록 <body>의 자식인 경우로 작성)
	
※ 'static'을 제외한 다른 설정의 경우, 추가적으로 위치를 지정하는 속성이
	필요하다.
	- 위치를 지정하는 속성 : [top], [left], [right], [ bottom]
※ 겹쳐진 영역의 순서는 [z-index]속성을 사용하여, 제어할 수 있다.
※ 상위포지션이란 : 상위요소(조상요소) 중에서 'static'을 제외한 
	포지션 속성이 설정된 가장 가까운 요소
	
	
■ 각도의 단위
	(1) deg : 육십분법/degree (360deg = 1회전) - ★
	(2) grad : 그레이드법 (400grad = 1회전)
	(3) rad : 호도법 (2π rad = 1회전)
	(4) turn : 회전 (1turn = 1회전)
	
■ 색상의 표기방법
	(1) 영문 색명으로 표현 : red, blue, .....
	(2) 색상코드로 표현 :  #565656, #FFF
		: 16진수로 된 6자리or3자리의 숫자로 구성
	(3) rgb() / rgba()의 값으로 표현
		background : rgba(255,0,0,0.5);
	(4) hsl() / hsla()의 값으로 표현
		background : hsla(360,100%,100%,50%);
